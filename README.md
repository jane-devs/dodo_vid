# Мониторинг стола по видео

Инструмент на Python 3.13 и YOLO для анализа видеозаписи: определяет моменты, когда человек появляется за столом, и считает статистику времени ожидания между посещениями.

---

## Установка

```bash
pip install -r requirements.txt
```

> Автоматически скачается веса `yolov8n.pt` (~6 МБ) при первом запуске.

---

## Использование

```bash
python main.py --video video1.mp4
```

### Выбор области интереса (ROI)

После запуска откроется окно с первым кадром видео.
Нарисуйте прямоугольник вокруг стола мышью и нажмите **Enter** или **пробел** для подтверждения.
Нажмите **Escape** для отмены (программа завершится с сообщением об ошибке).

---

## Логика детекции

- **Модель:** YOLOv8n, класс `0` (человек).
- **Проверка попадания:** для каждого bounding box вычисляется центр `(cx, cy)`. Человек считается «у стола», если центр попадает внутрь выбранного ROI.
- **Конечный автомат:** два состояния — `пусто` и `занято`.
- **Дебаунс:** состояние меняется только после `DEBOUNCE_FRAMES = 8` подряд идущих обработанных кадров с одинаковым сигналом — защита от ложных срабатываний.
- **События при переходах:**
  - `пусто → занято`: записываются события `подход` и `занято`.
  - `занято → пусто`: записывается событие `пусто`.
- **Оптимизация:** обрабатывается каждый второй кадр (`FRAME_SKIP = 2`); пропускаемые кадры захватываются через `cap.grab()` без декодирования.

## Результаты работы

| Артефакт | Описание |
|---|---|
| `videoname_timestamp.mp4` | Исходное видео с наложенным прямоугольником ROI: **зелёный** — стол пуст, **красный** — занят. В углу ROI отображается текущее состояние и метка времени `MM:ЧЧ`. |
| `videoname_timestamp.txt` | Текстовый отчёт: количество событий, число циклов занятости, среднее/мин./макс. время ожидания между посещениями. |

Консоль выводит сводную таблицу пар `(пусто → следующий подход)` и итоговые метрики.


---

## Известные ограничения

- **Нет разделения персонала и гостей** — любой человек в зоне ROI считается посетителем.
- **Один стол за запуск** — ROI задаётся вручную, мониторинг нескольких столов одновременно не поддерживается.
- **Чувствительность к освещению** — YOLOv8n может пропускать людей при резких перепадах яркости, бликах или низкой освещённости.
